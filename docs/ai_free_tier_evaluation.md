# 🧠 機器人 AI 辨識與對話功能免費額度評估報告

在重構本專案的 NSFW 圖片辨識、對話分析與錯字糾正等功能時，若引進 AI 技術，最核心的考量是**避免任何金錢開銷 (以免費額度為主)**。

以下針對現行主流的 AI API 提供免費額度評估，並推薦最適合本專案的「零成本」方案。

## 1. 🥇 首選推薦：Gemma 4 31B (最高的免費可用性)

### 📊 免費額度與限制
* **適用模型**: `gemma-4-31b-it`（支援多模態，可同時處理「文字」與「圖片」）。
* **免費額度**: 每分鐘 15 次 (RPM) / 每天 1,500 次 (RPD)
* **費用**: **0 元** (超出限制時會回傳 429 錯誤，但**絕對不會**自動扣款，除非您主動綁定信用卡並升級為付費 Pay-as-you-go 方案)。

### 💡 模型優勢
* **最高的免費可用性**：這是免費方案中每天給予最多請求數的模型。一天 1,500 次的對話量，對於中小型 Discord 群組來說已經相當充裕。
* **強大的生成與推理能力**：作為 31B 參數的密集模型，它的文字理解與生成能力也非常優秀，足以應付絕大多數的機器人閒聊、即時股價分析或指令任務。

### 🔄 透過多金鑰輪詢 (Key Rotation) 水平擴充額度
為了解決免費額度在高峰期被單一使用者打滿的問題，本專案實作了**金鑰自動輪詢與冷卻機制**：
* 您可以申請多個 Google AI Studio 帳號以取得多個免費 API Key，並全部設定在 `auth.json` 中的 `geminiApiKeys` 陣列或 `GEMINI_API_KEYS` 環境變數中。
* 機器人呼叫 API 時會自動輪詢使用這些 Key。如果其中一個 Key 遇到 429 限流或配額耗盡，系統會自動將該 Key 標記為 **5 分鐘冷卻**，並無縫切換到下一個可用金鑰。
* **效果**：藉由設定多組免費金鑰（例如 3 組金鑰即可將額度提升至 45 RPM / 4,500 RPD），不需要付任何費用，即可獲得高併發與大容量的 AI 服務支援。

### 💡 在本專案的應用場景 (零成本)
1. **圖片 NSFW 偵測（替代舊 TensorFlow 本地模型）**:
   * 當用戶發送圖片或執行 `!nsfw [圖片網址]` 時，將圖片的 Buffer 或 URL 傳給 Gemini，並在 System Instruction 中指示它：「分析此圖片是否包含成人內容(NSFW/Porn/Hentai)，請以 JSON 格式回覆安全等級百分比與描述。」
   * **好處**：完全免去本地端編譯 TensorFlow 的痛苦，機器人檔案大小可縮減數百 MB，並可自由升級 Node.js 到最新版本。
2. **智慧錯字糾錯與幽默吐槽**:
   * 目前專案只用語義死板的 `因該` 抓錯字。引入 Gemini 後，可對頻道內的所有聊天訊息進行輕量級審核，當偵測到錯別字時，由 Gemini 生成一段好玩、傲嬌或嘲諷的錯字指正回覆。
3. **AI 智慧聊天助理與即時股價分析**:
   * 允許成員以 `!bobo [對話內容]` 或 @ 機器人進行對話，調用 Gemini 進行聊天。當提及股票代號/暱稱時，結合 `yahoo-finance2` 動態查詢即時數據提供精準分析。

## 2. 🥈 備選方案：Hugging Face Serverless Inference API

Hugging Face 提供免費用於測試與開發的 Serverless 推論服務，可直接調用開源的圖像分類模型。

### 📊 免費額度與限制
* **適用模型**: 眾多開源的 NSFW 圖像分類模型（例如 `Falconsai/nsfw_image_detection` 或 `AdamC3/vit-nsfw-classifier`）。
* **費用**: **0 元** (只需註冊 Hugging Face 免費帳號並取得 Access Token)。
* **限制**: 
   * 沒有嚴格的 RPD/RPM 限制，但若模型太久沒有人調用，會遇到冷啟動（Cold Start）需要等待數十秒載入。
   * 僅能做單一任務（例如只能做 NSFW 圖像分類，無法進行文字對話或錯字糾錯）。

### 💡 在本專案的應用場景
* 僅單純用於 NSFW 圖片預測。我們可以直接發送 HTTP 請求（傳入圖片 binary），Hugging Face 會回傳 `[ { "label": "normal", "score": 0.98 }, { "label": "nsfw", "score": 0.02 } ]` 等 JSON 數據。
