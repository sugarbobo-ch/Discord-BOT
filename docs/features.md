# 🤖 BoboBot 機器人功能與指令大全

本文件詳細列出 BoboBot 目前支援的所有功能與指令。

機器人在伺服器中的預設指令前綴為 `!` 或 `！`。

---

## 1. ⚙️ 自訂指令系統 (Custom Commands)
伺服器成員可以動態為機器人新增、編輯或刪除指令。這些指令將保存在本地的 SQLite 資料庫（`config/bobo.db`）中，不同伺服器之間的指令不會互相干擾。

* **新增指令**
  * `!add [指令名稱] [回覆內容]`
  * 範例：`!add 嗨 你好啊！` (之後輸入 `!嗨` 機器人會回覆 `你好啊！`)
* **編輯指令**
  * `!edit [指令名稱] [回覆內容]`
  * 範例：`!edit 嗨 滾！`
* **刪除指令**
  * `!remove [指令名稱]`
  * 範例：`!remove 嗨`
* **查詢目前伺服器可用指令**
  * `!list` 或 `!help`
  * 顯示基本說明與使用指引（使用 Embed 呈現）。
* **關鍵字搜尋自訂指令**
  * `!大全 [關鍵字]`
  * 範例：`!大全 測試` (列出所有名稱包含「測試」的自訂指令)
* **初次引進設定**
  * `!reset server`
  * 初始化當前伺服器的自訂指令檔（在機器人首次被邀請入群時，可用於防範未經授權的邀請連結分享）。

### 💡 進階功能：變數替換
自訂指令內容如果包含 `{}`，則可以使用變數帶入功能。
* 範例：`!add 搜尋 https://www.google.com/search?q={}`
* 使用：`!搜尋 Discord` -> 機器人回覆：`https://www.google.com/search?q=Discord`
* 如果傳入多個變數，機器人會依序將 `{}` 替換為輸入的文字；若變數不足，剩餘的 `{}` 將重複填入第一個參數。

## 2. 🖼️ 自訂隨機圖片/媒體系統
可以建立一個指令，使其每次被觸發時隨機從專屬資料夾中挑選一張圖片或媒體發送。

* **初始化隨機圖片指令**
  * `!add [指令名稱] 隨機圖片`
  * 範例：`!add 貓貓 隨機圖片` (這會在 `assets/images/貓貓/` 建立專屬圖片資料夾)
* **新增圖片到特定隨機指令**
  * `!addimg [指令名稱] [圖片 URL]`
  * 範例：`!addimg 貓貓 https://example.com/cat.jpg` (機器人會下載圖片到伺服器並加入隨機清單，非 NSFW 頻道會觸發 AI 安全審查)
* **刪除特定指令中的圖片**
  * `!delimg [指令名稱] [圖片檔案名稱]`
  * 範例：`!delimg 貓貓 cat.jpg` (將該圖片從資料夾與索引中移除)
* **觸發隨機圖片**
  * `![指令名稱]`
  * 範例：`!貓貓` (隨機發送一張貓貓資料夾底下的圖片)

*備註：代碼中亦有支援 `隨機媒體`（對應 `assets/media/` 資料夾）的邏輯。*

## 3. 🎟️ 隨機抽獎系統 (Lottery)
在頻道中發起抽獎，並隨機抽取出中獎者。

* **發起抽獎**
  * `!開始抽獎 [活動標題] [時間(分鐘)]`
  * 範例：`!開始抽獎 送飲料 10` (發起一個 10 分鐘後截止的「送飲料」抽獎。時間不填預設為 5 分鐘)
* **參加抽獎**
  * `!抽獎` (伺服器成員在活動時間內於該頻道輸入即可參加，舉辦人無法參加)
* **查看抽獎名單與說明**
  * `!抽獎名單` (顯示抽獎狀態、截止時間與當前已參加的成員名單)
* **開獎**
  * `!開獎 [中獎人數]`
  * 範例：`!開獎 3` (隨機抽取出 3 名中獎者，並以帶有頭像的 Embed 公布。限舉辦人操作，且必須在截止時間後才可執行)
* **結束/關閉抽獎**
  * `!結束抽獎` (僅限舉辦人操作，用以刪除該頻道的抽獎狀態與紀錄)
  * `!強制結束抽獎` (強制清除該頻道的抽獎狀態)
* **指令說明**
  * `!抽獎指令` (顯示抽獎系統的所有指令介紹)

## 4. 📝 點名系統 (Roll Call)
簡單的群組點名與簽到功能。

* **開始點名**
  * `!開始點名 [點名活動標題]`
  * 範例：`!開始點名 10:00 例會`
* **成員點名簽到**
  * `!點名 [備註]`
  * 範例：`!點名 在此` (記錄成員點名時間與備註)
* **查看點名清單**
  * `!點名清單` (顯示已簽到的名單與點名時間)
* **結束點名**
  * `!結束點名`
  * *機制：為防止惡意操作，需要有 3 個人投贊成票（即 3 個不同的成員執行此指令），才會正式關閉該伺服器的點名活動。*

## 5. 📌 訊息暫存系統 (Keep)
使用者可在機器人記憶體中暫存特定文字，類似個人剪貼簿。

* **暫存訊息**
  * `!keep [文字內容]`
  * 範例：`!keep 晚點要去買晚餐` (會儲存該訊息，每位使用者最多儲存 10 筆，超過會擠掉最舊的)
* **查看個人暫存清單**
  * `!keeplist`
  * 顯示該使用者儲存的所有 Keep 訊息。
  * *注意：此資料目前保存在機器人記憶體中，當機器人重啟後資料將會消失。*

## 6. 🔞 NSFW 與實用工具指令
專為 ACG 社群設計的相關查詢與外部連結生成功能，並內建對特定網站的自動預覽抓取機制。

* **外部連結快捷指令**
  * `!god [車號]`、`!nhentai [車號]` 或 `!god [車號]`：自動生成 nhentai 的作品網址。
  * `#[車號]` (快捷格式，必須為 **7 個字元** 且為 `#` 開頭，後接 6 位數字，例如 `#228922`)：自動轉換並發送對應的 nhentai 連結。
  * `!pixiv [作品ID]`：自動生成 Pixiv 的作品網址。
  * `!wnacg [車號]`：自動生成 wnacg 的作品網址（會先偵測該頻道是否為 Discord NSFW 頻道，若非 NSFW 頻道則會阻擋並警告）。
  * `!搜圖 [圖片網址/上傳圖片/回覆圖片]`：使用 SauceNAO 搜尋圖片的詳細資訊（標題、畫師、來源連結、相似度等）。

### 🕵️‍♂️ 連結自動預覽生成 (NSFW Embed Scrapers)
當使用者在 **NSFW 頻道**發送特定網站的連結時，機器人會等待 3 秒鐘。如果 Discord 沒能為該連結載入縮圖預覽，機器人會自動抓取該作品的資料並發送精美 Embed：
1. **E-Hentai / ExHentai**：利用 E-Hentai JSON API 查詢，將標籤拆分為畫師 (Artist)、社團 (Group)、女性/男性屬性等精細排版。
2. **Wnacg 紳士漫畫**：抓取 HTML，解析出標題、畫師、分類和標籤，並處理相對路徑或無協定的封面圖。
3. **18Comic 禁漫天堂**：為防止域名遭 ISP 阻擋或 DNS 污染，會依序輪詢多個可用鏡像（如 `18comic.vip`, `18comic.ink`, `jmcomic.me`, `18comic.org`），抓取成功後解析並繪製 Embed。
4. **Happymh 嗨皮漫畫**：直接抓取網頁容易遇到 403 Forbidden 擋牆。機器人在直接抓取失敗時，會**自動透過 Google Translate 代理服務**（`translate.google.com`）作為 Proxy 重新抓取網頁內容以成功獲取漫畫資訊與封面。

## 7. 💬 自動觸發互動功能 (Easter Eggs)
機器人在背景自動監聽訊息並觸發的互動功能。

* **複讀機功能 (Repeat)**
  * 當同一個頻道中，有連續 5 次一模一樣的訊息發送時（且在 10 分鐘內），機器人會自動「複讀」發送一次相同的訊息。
* **群發廣播指令**
  * `!send [頻道ID] [訊息內容]`
  * 機器人會以自身身份，將後續的訊息內容群發到指定的頻道中（常用於管理員後台廣播）。

## 8. 🧠 Gemini AI 整合功能
本專案整合了 **Google GenAI SDK**，採用 `gemma-4-31b-it` 模型，為機器人注入更靈活的智慧對話、圖片分析與即時股價查詢能力。

### 💬 AI 趣味對話 (`!bobo`)
* **觸發方式**：`!bobo [對話內容]` 或直接 @ 機器人。
* **性格設定**：說話風格幽默、風趣且親切友善，講話隨性，偶爾帶點好玩的小吐槽（焦糖波波為開發者）。
* **頻道記憶 (Context Memory)**：會根據 `auth.json` 中的 `chatMemoryLimit` 自動載入該頻道最新 $k$ 筆聊天紀錄。時間越近的文字權重/熱度越高（最新一筆為 1.00），並附加相對時間（如「30秒前」），讓 AI 在回覆時能融入當前聊天環境。
* **多模態支持**：支援上傳圖片或提供圖片網址，AI 會一併讀取圖片與文字進行分析。
* **靜態防護與頻率控制**：具有個人 5 秒的冷卻時間，並具備提示詞注入防禦（Prompt Injection Defense），若試圖刺探 Prompt 或底層環境變數會被幽默拒絕。
* **LaTeX 符號清理 (`cleanLatexSymbols`)**：自動清除 Gemini 回覆中常見的 LaTeX 數學公式與轉義符號（如 `\rightarrow` 轉為 `→`，移去 `$` 符號），並且自動將不支援的四級標題轉換為 H3 (`###`)，確保在 Discord 呈現時美觀且易讀。

### 🔄 API 金鑰自動輪詢與容錯 (Key Rotation)
為了確保 AI 服務的可用性，系統支援配置多組 Gemini API Key。當呼叫遇到 `429` (Rate Limit)、`RESOURCE_EXHAUSTED`、`403` 或 `401` 等錯誤時：
- 該 Key 將會自動進入 **5 分鐘的冷卻期**。
- 系統將會自動輪詢切換到其他可用的 API Key 重試。
- 若所有 API Key 都在冷卻中，則會退而求其次使用冷卻到期時間最早的 Key。

### 📈 即時股票分析工具 (Stock Analysis)
當使用者在 `!bobo` 對話中提及任何股票關鍵字、代號（例如台股 `2330`、美股 `AAPL`）或是常見股票暱稱（如 `發哥` 代表聯發科、`公公` 代表鴻海、`西瓜` 代表新光金）時：
1. **預先偵測與即時查詢**：系統會透過正則與 `detectStocksWithAI` 提取代號，並調用 `yahoo-finance2` 查詢最新真實股價、漲跌幅、成交量、EPS、本益比等指標，並將資料進行 **60 秒記憶體快取** 以防請求氾濫。
2. **分析師人格切換**：查得股票資料後，系統會將真實數據注入 Gemini 提示詞中，並將 System Prompt 切換成 **專業投資分析師與基金經理人** 角色。
3. **使用者隔離 (Isolation)**：若聊天頻道有多人對話，系統會特別在 System Prompt 中區分不同的說話者，防止將 A 使用者的股票配置或話題套用到 B 使用者身上。
4. **超時容錯快取 (Cache Fallback)**：當呼叫 Gemini 生成詳細產業報告逾時 (Timeout) 或失敗時，Bot 會直接回傳已經查到的即時股價與財務資訊，避免使用者空等。

### 🔞 NSFW 圖片上傳安全檢測
* **機制**：當在**非 NSFW 頻道**使用 `!addimg` 新增隨機圖片時，系統會調用 Gemini 多模態分析圖片是否包含裸露、暴力、血腥等敏感內容。若判定為 NSFW，機器人會立即刪除檔案、拒絕新增並說明原因。

### ✍️ AI 錯字糾正與吐槽
* **機制**：系統背景監聽常見錯字，包括 `'因該'`（應該）、`'以經'`（已經）、`'部會'`（不會）、`'絕得'`（覺得）、`'在一次'`（再一次）。當偵測到錯字且未在冷卻時間內（15 秒），會調用 Gemini 產生一句 50 字以內幽默風趣的句子來提醒使用者打對字。若 API 失敗或冷卻中，則自動降級使用本地預設的吐槽。

## 9. 🐦 Twitter/X.com 連結自動修復與伺服器設定 (Auto Fix)
* **自動修復機制**：當任何使用者在頻道中貼出 `x.com` 連結時，機器人會自動於背景等待 3 秒鐘。若偵測到 Discord 沒有成功為該連結產生 embed 網頁預覽（`embeds.length === 0`），機器人會自動把 `x.com` 替換為 `fixvx.com` 並在同頻道發送，確保群組內成員都能正常觀看圖片/影片預覽。
* **伺服器管理設定**：
  * 管理員可在伺服器中使用 `!設定`、`!setting` 或 Discord 斜線指令 `/設定`。
  * 機器人會回傳帶有 Button (按鈕) 互動介面的訊息，允許管理員點擊開啟或關閉該伺服器的 Twitter 自動置換功能，設定狀態會持久化儲存於 SQLite 的 `settings` 資料表中。
